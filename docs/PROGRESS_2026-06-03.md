## 1. 로그인 개선

### 구글 로그인 속도 (프론트 최적화, 무비용)
- **원인**: 로그인 완료 판정이 백엔드 프로필 조회까지 블로킹. `useUserProfile`이 수동 훅이라 컴포넌트마다 `onAuthStateChanged` 구독 + `/api/ai/profile` 독립 호출(캐시·dedup 0) → login·dashboard에서 "로딩 중" 반복.
- **조치**: `web/hooks/useUserProfile.ts`를 `useQuery`(key=`["user-profile",uid]`, staleTime 5분)+`useMutation`으로 재작성. 반환 인터페이스 동일 유지 → 호출처 6곳 무수정. 프로필 1회 fetch를 전역 공유.
- 콜드스타트(min=0) 자체는 인프라 비용 거부로 미적용.

### 카카오 로그인 (custom token)
- `screener/api/auth_routes.py`: `POST /api/auth/kakao` {code, redirect_uri} → httpx로 카카오 token/user → `create_custom_token(uid="kakao:{id}", {provider, email})`. 키 없으면 503. `main.py` 등록 + `middleware.py` PUBLIC 경로.
- `web/lib/auth-actions.ts`: `startKakaoLogin`/`completeKakaoLogin`. `web/app/(auth)/login/kakao/callback/page.tsx` 신규. login 버튼 활성화.
- env: `KAKAO_REST_API_KEY`(백) / `NEXT_PUBLIC_KAKAO_REST_API_KEY`(프), 선택 `KAKAO_CLIENT_SECRET`.

### 이메일/비밀번호 회원가입 + 어뷰징 방어
- `auth-actions.ts`: `signUpWithEmail`(가입+인증메일), `signInWithEmail`, `sendPasswordReset`, `resendVerification`, `looksDisposable`.
- `login/page.tsx`: 소셜 + 이메일 폼 3모드(signin/signup/reset). 회원가입은 일회용메일 차단·비번8자+·확인일치, 가입 후 signOut+인증안내. 미인증 로그인 시 경고 토스트 + **인증 메일 재발송 액션**.
- **어뷰징 핵심**: 트라이얼은 `email_verified` 필수(`start_trial`·`can_start_trial`) → 미인증 이메일 계정은 free만, 14일 트라이얼 불가.
- `security.py`: `DISPOSABLE_DOMAINS` +28·`KEYWORDS` +7. `check_trial_abuse` IP 규칙 강화 — 24h 내 다른계정 1개 **또는** 30일 내 다른계정 2개 이상이면 차단(30일 단일 쿼리로 통합).
- 운영 트라이얼은 카드(LS) 기반이 기본 = 카드 fingerprint 근본 차단(유지).

## 3. 종목 분석 화면 전환 시 백그라운드 지속

- **문제**: `AnalyzeView`가 분석 상태를 로컬 `useState`로 들고, useEffect cleanup `ac.abort()`로 언마운트 시 SSE 중단 → 분석(30~90초) 중 화면 이동하면 멈춤.
- **해결**: `web/store/analysisStore.ts`(zustand) 신설. ticker별 `AnalysisRun` + AbortController(모듈 Map). `start()`가 `apiStream`을 store에 누적 — 컴포넌트 수명 무관. `AnalyzeView`는 구독+렌더만(로컬 상태/SSE useEffect 제거). 화면 떠나도 fetch 계속→재진입 복원. 페르소나 전환 시 기존 abort+새 실행.
- 한계: **탭을 닫으면** 끊김(서버 잡 아니면 불가피). 필요 시 백엔드 job+폴링으로 확장 가능.
