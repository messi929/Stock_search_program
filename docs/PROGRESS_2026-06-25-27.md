## 0. 맥락 — 왜 마케팅인가

가입자 ≈ 0 = 유입/전환 문제가 아니라 **0→1 문제**. 1차 목표 = 진짜 첫 사용자 **10~50명**.
채널 = **스레드(Threads) 단일**(팔로워 0이어도 알고리즘 노출, 익명 운영 적합). 창업자 얼굴 X, 예산 ≈ 0.
포지셔닝 = 법적 제약(추천·목표가 금지)을 **무기로** → "추천 안 함, 검증 + 반대의견(Contrarian)".

2. **종목글 "퀄리티 별로" 수정** (예시: 숫자 0개 + "뭔가 움직임/평소와 다른 패턴" 헛글 + "AI한테 물어봤더니" 프레이밍):
   - **프롬프트 전면 개편**: 실수치 **≥2개 숫자 그대로 인용 강제** + 의미 부착, "뭔가 움직임/패턴/눈에 띈다" 등 헛소리 명시 금지, **"AI한테 물어봤더니/AI에게 물어보니" 프레이밍 절대 금지**(글의 주어 = AI 아닌 '수치').
   - **데이터 가드** `_numeric_fact_count() < 2`(`MIN_NUMERIC_FACTS`)면 생성 스킵 — 빈 스냅샷 헛글 차단.
   - **모델 Haiku → Sonnet 기본 승격**(`MARKETER_MODEL` 상수, `=haiku`로 회귀 가능). 비용 ~5원 → ~22~35원/편, 하루 ~95원.

   - 검증: 4포맷 전부 dry-run(삼성전자) — `-5.30%/ROE 17.2%/PER 27.4/PBR 4.72/거래량 1.2배/52주 -6.3%/RSI 55` 줄줄이 인용, 면책·AI프레이밍·추천 전부 0.

3. **502 핫픽스 + 종목명 입력 지원** (직후 발견): 위 데이터 가드 도입 후, 관리자가 **종목코드 대신 종목명**(예: `HD일렉트릭`)을 입력하면 `build_instant_snapshot`이 티커로만 조회 → 전부 스킵 → 빈 결과 → 라우트가 **502 "AI 응답 없음"**(오해 소지)으로 크래시.
