## 3. 진단 — 적자를 만드는 3개 구멍

### 🕳️ A. Quota가 enforce되지 않음 (`api/routes/ai.py`)
```python
PLAN_LIMITS = {
  "free": {"analyses": 20, ...},   # 정의는 됨
  "pro":  {"analyses": -1, ...},   # 무제한
}
```
- `PLAN_LIMITS`는 **`/api/ai/usage` 화면 표시에만** 사용. 분석 진입점(`/analyze`)은 **횟수를 안 막음**.
- 결과: **Free조차 21회째가 차단 안 됨**. 무료 사용자가 Opus를 무한 호출 가능 = 가장 큰 구멍.
- Pro는 설계상 무제한(-1)이라 어뷰징/헤비유저 적자에 무방비.

### 🕳️ B. Opus 편중 (비용 76%)
- Strategist가 4개 분석 종합 + 페르소나 적용으로 Opus 사용. 건당 294원.
- input도 큼(종합 컨텍스트). 캐싱 미작동으로 input 절감 미실현.

### 🕳️ C. 캐싱 미실현
- system 프롬프트 `cache_control: ephemeral` 설정은 있으나(`utils/claude_client.py`) 실측 cache_read ≈ 0.
- 응답 캐싱(`utils/cache.py`)은 **메모리 기반**(Cloud Run 재배포 시 소멸) + 분석 경로 연결 불명확.
- 동일 종목 반복 분석에도 매번 풀 호출.

> 부수 발견: Firestore가 `agents.<name>.<field>` **점 포함 평면 키**로 저장됨(`set(merge=True)` 특성). `/api/ai/usage` 조회 코드가 nested로 읽으면 0이 나올 수 있음 → **점검 필요**.

## 4. 해결 설계

### A. Quota Enforcement (최우선 · 적자 구멍 차단)
분석 진입점(`/analyze`)에서 **호출 전** 당월 사용량 체크 → 초과 시 차단(402/429).

| 플랜 | 분석/월 | 검증/월 | 근거 |
|---|---|---|---|
| Free | 20 (실차단) | 10 | 기존 의도값. 원가 20×389=7,780원 → 무료 미끼비용 상한 |
| Pro | **공정사용 100회** | 50 | 99% 사용자 무영향. 100×389=38,900원이지만 캐싱·모델최적화 후 ~15,000원 |
| Premium | 300 | 150 | ARPU 높아 여유 |

- 구현: `_check_and_reserve_quota(uid, tier, kind)` — Firestore 트랜잭션으로 당월 카운트 조회 후 차단. cost_tracker가 이미 일별 기록 → 월 합산 재사용.
- "무제한" 마케팅 문구 → **"공정사용 정책 내 무제한"**(약관 명시). 상한 도달 시 안내 UI.

### B. 변동비 절감 — Strategist (최대 효과)
목표: 389원 → 120~160원.

1. **모델 전환 ✅ 적용(2026-06-02)**: Strategist Opus 4.7 → **Sonnet 4.6** + max_tokens 1500→2560.
   - **A/B 검증**(3종목 × 2모델, `scripts/strategist_ab.py`): 비용 **76% 일관 절감**(Opus ~300원 → Sonnet ~72~85원). 품질은 종합·페르소나·법적안전 **동등 이상**(Sonnet summary가 오히려 더 풍부).
   - max_tokens 1500이면 Sonnet 서술이 길어 마지막 필드(alert_conditions)가 잘림 → 2560 상향으로 graham 등 최장 케이스도 alert×4 채움 확인.
   - 결과: 딥다이브 **389원 → ~175원**(55%↓), 손익분기 **월 22회 → 월 48회**.
2. **Prompt caching 정상화** (미적용): 페르소나 system 프롬프트를 안정적으로 캐시(cache_read 90% 단가) → input 비용 추가 절감. A/B 로그상 `cache_w`만 크고 `cache_r=0` 확인.
3. **응답 캐싱 영속화 ✅ 적용**: ResponseCache를 L1(메모리)+L2(Firestore `ai_response_cache`, TTL 6h) 2단으로. 캐시 키가 messages(현재가·MA·"기준 시각") 해시라 **데이터 갱신 시 키 변경 → 자동 신선도 보장**(동일 스냅샷에서만 hit). 콜드스타트/재배포 후에도 동일-데이터 재분석 시 API 0. Firestore I/O는 to_thread 비블로킹 + graceful(장애 시 캐시 없이 진행).

> 점검 포인트: A/B 중 `opus in=6 out=20` 미세 호출이 조합당 1회 관찰 — 그래프 내 중복/빈 호출 가능성(비용 미미).

### C. 플랜·가격 재설계
- **연 구독 도입**(`VARIANT_YEARLY` 코드 이미 존재): 99,000원/년(2개월 할인). 수수료 14%→7.7%, 이탈↓, 선결제 현금흐름.
- **Premium 활성화 검토**: 고가일수록 고정수수료 비중↓ + 헤비유저 수용.
- VAT 세금 별도 적용(LS 설정).
