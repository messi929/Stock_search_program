## 1. ① eval 측정 하네스 (`evals/`)

기존엔 출력 *품질*을 점수로 보는 도구가 없었음(`tests/regression/test_60_cases.py`=실행여부,
`scripts/legal_check.py`=코드 정적스캔). 실제 출력물을 다차원 점수화하고 회귀 비교한다.

- `dataset.py` 골든 케이스(default 11/smoke 1/full 60, persona_diff 그룹 포함)
- `scorers.py` legal·completeness·numeric·persona_diff(결정론·무료) + LLM-judge
- `runner.py` 실행→채점→스코어카드→baseline diff(전체 출력 덤프 저장)
- `tests/test_evals.py` 결정론 채점기 단위테스트(API 불필요)

```bash
py -m evals.runner --judge --label baseline
#   (개선 적용)
py -m evals.runner --judge --label after --baseline evals/results/baseline.json
```

### judge는 반드시 Sonnet (작업 중 발견)
초기 Haiku judge는 reasoning/data를 **일괄 2점으로 포화**시키고 자기검증 문장을
환각으로 오판해 판별력이 없었음. Sonnet으로 교체하니 정상 변별. 또 judge에
**페르소나 철학(personas/*.md) 주입** → 그레이엄 'PER<15' 같은 표준기준 오판 방지.
(`AXIS_JUDGE_MODEL=haiku`로 강제 가능)

## 2. ③ Extended Thinking (`utils/claude_client.py`)

복잡한 종합·검증 노드의 추론 품질 향상.
- `thinking_budget` 배선(base→client). SDK 0.43.0이 네이티브 미지원 → **`extra_body`로
  주입**(버전무관, 실호출 검증). max_tokens 가산·temperature 1.0 강제·캐시키 분리.
- Strategist(`STRATEGIST_THINKING_BUDGET=3200`) + Validator(`VALIDATOR_THINKING_BUDGET=1600`).

## 3. ② 해석 검증 (`validator.py` → `strategist.py`)

Validator가 수치만 검증하던 것을 **Analyst 해석의 타당성**까지 감사하도록 확장
(기존 contrarian 호출에 통합 = 추가 호출 0). `interpretation_audit`로 과신·방향비약·
톤불일치·데이터모순 탐지 → Strategist가 종합 시 보정. `AXIS_INTERP_AUDIT=0`로 비활성.

- **실측 효과**: blackrock:005930에서 후속 분석이 "'외국인 3일 연속 순매수'가 긍정으로
  서술됐으나 리서치는 상위에 없다고 명시 — 상충" / "검증가와 2.05% 괴리"를 **스스로 지적**.
  (전에는 그대로 긍정 단정) judge도 "Validator 감사 지적을 실제로 반영"으로 인식.

## 4. ④ 섹터 상대 밸류에이션 (`analyst.py`)

"업종 평균"을 LLM이 추측(환각 위험)하지 않도록, 시장 스냅샷에서 **섹터 중앙값
PER/PBR/ROE + 백분위**를 결정론적으로 산출해 앵커로 주입. `valuation_judgment`·
`peer_avg_per`의 근거가 데이터에 고정. `AXIS_SECTOR_STATS=0`로 비활성.
