## 1. 밸류에이션 밴드 백필 — 0건 저장 근본원인 해소

전날(6/29) 밴드 코드/잡/스케줄러는 배포됐으나 **데이터 0건**이었다. "야간점검 탓"으로
기록돼 있었으나 **오진**이었다.

### 진단 (근본원인)
- 낮(월 19:52 KST)에 돌려도 20초 만에 가드 조기중단 → 야간 문제 아님.
- 로컬 재현: pykrx `get_market_fundamental_by_date`가 과거 날짜도 **빈 (0,0) DataFrame**.
  `get_market_ohlcv_by_date`는 정상 → 네트워크/이미지 문제 아님(교차확인).
- **진짜 원인**: KRX가 펀더멘탈(PER/PBR/EPS)에 **로그인 인증을 요구**하도록 변경. pykrx≥1.2.8은
  `KRX_ID/KRX_PW` env로 로그인하는데, `axis-fundamental-band` 잡 env에 그 시크릿이 **미주입**
  (FIREBASE/ANTHROPIC만) → 매 호출 빈 응답 → 가드가 정상 동작해 0건.

### 수정 (커밋 `475ebb6`, `403ec94`, `965d550`)
- 잡 + `deploy-fundamental-band-job.sh`에 `KRX_ID=krx-id:latest,KRX_PW=krx-pw:latest` 주입
  (시크릿은 all-of-asset에 **이미 존재**) + SA `secretAccessor` 바인딩.
- 인증 fetch가 종목당 ~10~12초(5년)로 무거워 5년·300종목이면 30분 타임아웃 초과 발견 →
  기본값 **3년·sleep 0.5초**, 타임아웃 **60→80분**, 재시도 0.
- **N(50)종목마다 증분 저장**(`--flush-every`) — 끝에서 일괄 저장하던 구조라 중도 타임아웃 시
  전손이던 위험 제거(직전 flush까지 보존). `save_fundamental_band`은 종목당 덮어쓰기라 재호출 안전.
- **첫 백필 성공: 288/300 종목 적재**(12종목은 표본부족/신규상장, 정상). 소요 ~56분.

## 2. 새벽 미국장 브리핑 — 화~토 + 신선도 가드 + 종목글 자동생성 분리

### 발단(JEON)
- "아침 미국 브리핑은 화수목금토만 가능하지 않나" + "왜 월요일에 안 열린 미국장을 브리핑하나".

### 진단 (둘 다 맞음)
- 발행 경로 = `axis-threads-daily` 잡(`jobs.daily_threads_content`), `30 7 * * 1-5`(월~금).
- 미국은 주말 휴장 → 미 월요일장은 **22:30 KST 개장**. 그래서 KST **월요일 07:30** 아침엔
  직전 미국 세션이 없음(금요일이 마지막). FDR이 금요일 묵은 종가를 줘 "[월 새벽 미국장]"으로
  **오발행**. 게다가 정작 금요일 세션을 다룰 **토요일 아침이 스케줄에서 누락**(1-5).
- 월요일 국내장 길잡이는 **일요일밤 주말 브리핑**이 이미 담당 → 월 아침 US 브리핑은 중복.

### 수정 (커밋 `a22439c`, `22e016c`)
- 스케줄 `1-5` → **`2-6`(화~토 07:30 KST)** — 간밤 미국 세션이 있는 아침만.
- `agents/briefing.py` **신선도 가드**(`us_session_is_stale`): 미국 **주식** 지표
  (sp500/nasdaq/dow/sox/vix) 최근 종가일이 KST 기준 2일 이상 묵으면 생성 생략 → 월·일 아침·
  **미 공휴일 직후** 자동 스킵(미 종가일은 날짜변경선상 KST보다 하루 뒤라 신선=diff 1).
  ⚠️ `usdkrw`(24/5 FX)는 월 아침 이미 당일 날짜를 가져 가드를 뚫으므로 **FX 제외**.
- 종목글(`--stocks`)을 **자동 생성에서 분리**(스케줄 args `--stocks 0`) — JEON 수동 운영.
  생성 코드·관리자 UI는 유지(수동 실행 시 `--stocks N`).
- 검증: 07:30 시나리오 — 월=생략 / 토(금세션)=생성 / 화(월세션)=생성.

## 3. 종목글 '앞으로 볼 것' 방향성 페이로드

### 발단(JEON)
- "마케팅 글 호응이 떨어진다. 종목글은 어디서 사야 할지/앞으로 뭐가 좋을지 **방향성**이어야".

### 진단
- 하네스가 `결론(사라/팔라) 빼고 한 끗만`을 강제 → **페이로드를 매번 보류**(독자 "그래서 뭐?").
  메인 포맷 contrarian은 과거 회의 중심(미래 방향과 반대). legal(추천/목표가 금지)이라도
  **조건부·관찰형 미래 방향**은 허용된다.

### 수정 (커밋 `e6e6c6c`, `4b33918`, `ebb41f5`, `73b28bf`)
- `ThreadsPost`/`EditedPost`에 `watchpoints` **필수 필드** → `assemble_post`가
  **'앞으로 볼 것 ①②③'** 블록 렌더. `_CORE_RULES`에 '판단 보류 금지 → 관찰 포인트 마무리'
  (수급/추세 조건·**지지/저항 관찰 수준**(매수가 아님)·실적 조건). 편집 루브릭 7축(방향성).
- **결정론적 가드**: watchpoints 비거나 500자 초과면 자기언급 가드와 묶어 1회 재편집
  (각 항목 35자·전체 480자·매수가/목표가/손절가 금지).
- **포맷별 정합**: watchpoints는 **contrarian·technical·curiosity 강제**, **cta는 면제**
  (댓글 유도가 마무리). curiosity 가이드는 '호기심 → 앞으로 볼 것(무엇을 지켜보면 풀리는지)'로 정렬.
- briefing/weekend_briefing은 같은 스키마라 watchpoints=[] 명시(관전포인트는 본문 ③).

### 검증 (prod rev 00128, dry-run)
- 올릭스(contrarian) 440자·오름테라퓨틱 426자 — 둘 다 ≤500, '앞으로 볼 것' 블록·관찰형·금지표현 0.
- 오름테라퓨틱은 **백필된 밸류에이션 밴드(PBR 2.57~21.75·중앙값 8.81·EPS 저점권)** 실제 인용.
- curiosity 356자=블록 O(호기심→관찰), cta 260자=블록 X·댓글 유도 마무리.
