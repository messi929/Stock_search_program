## 1. 데이터 페르소나 prod 안정화 (event/korean/macro)

대고객 오픈 인증 스모크에서 korean/event 페르소나가 prod에서 JSON 검증 실패로
"오류가 발생했습니다"를 반환하는 것을 발견. 추적 과정:

1. **truncation**: korean(3000)/event(3500) max_tokens 초과로 JSON 잘림 → 상향(f1c0d6f).
2. **캐시 마스킹**: max_tokens를 올려도 응답 캐시 키가 `(model,system,messages)`뿐이라
   옛 잘린 응답이 히트(2.5s 캐시히트로 발견) → 캐시 키에 max_tokens 포함(edc0ccd).
3. **event 스키마 견고화**: 8블록 중첩 스키마를 모델이 flaky하게 누락/타입오류 →
   default+사후보정(8cfac1d) + 최종 실패 시 graceful 강등(ffad066).
4. **스키마 단순화**(핵심): 프론트(EventAnalystCard)가 실제 쓰는 필드만 평탄·관대한
   `_EventLLMOutput`으로 받고 코드가 `EventAnalystResult`로 조립(`_assemble`).
   검증 실패 사실상 0(01975f1).
5. **점수 채움 강제**: 단순화 후 모델이 4차원 점수를 0으로 비워 항상 Refused →
   프롬프트 예시를 현실값+강한 지시로(4a32745).
6. **🚨 진짜 루트 원인**: 위 수정이 prod에 반영돼도 점수 0 지속 → `.gcloudignore`의
   `*.md`(gitignore 재귀 매칭)가 **personas/*.md를 Cloud Build 업로드에서 제외**하고
   있었음. event/korean/macro는 시스템 프롬프트가 오직 .md에서 와서 prod에서 빈
   프롬프트로 동작(strategist는 파이썬 상수라 가려짐). `!personas/*.md` 복구(2c8cbf5).
   검증: `gcloud meta list-files-for-upload | grep personas` 6개 .md 확인.

**결과**: prod 6 페르소나 전부 정상. event 4차원 점수 채워지고 Full Analysis 동작
(NVDA 8.7/RKLB 8.6/삼성전자 8.7). 상세 함정: 메모리 [[axis-production-cors]],
[[project_harness_quality]].

## 2. 이전 분석 카드 (analyze 페이지)

종목 진입 시 **차트 하단**에 직전 strategist 분석의 핵심을 보여주고, 그 아래에서 새
분석을 진행하는 원래 의도 구현(기존 /history 전역 목록과 별개).
- 백엔드: 이력에 entry_points/exit_points 저장 + `GET /api/ai/history/latest?ticker=`.
- 프론트: usePreviousAnalysis + PreviousAnalysisCard(분석일·당시가→현재 변화·1/2/3차
  진입·손절/익절·당시요약). 직전 분석 없으면 렌더 안 함.

## 3. 종목 발견 전면 고도화

직관성 개선 4방향 모두 반영:
- **접근성**: 사이드바 🧭 종목 발견 + 전용 `/discover`(DiscoverView).
- **가이드 입력**: 카테고리 프리셋(가치/성장/배당/모멘텀/역발상).
- **US 지원**: 시장 토글 KR/US/전체(`_gather_context` 멀티 스냅샷).
- **반복 정제**: '다른 종목 더 보기'(기존 제외 재요청).
- **결과 풍부화**: buy_score·PER/PBR/ROE·수급·테마 결정론 주입.
- briefing: `/api/ai/briefing` 공개경로 추가(X-Briefing-Key 보호).

## 4. 관심종목 구조 정리 + 모니터링 강화

추가는 인라인(분석/발견/검색)으로 충분 → 사이드바 '관심 종목'(추가 페이지) 제거.
관심종목을 '지켜보는 곳'으로 — 대시보드에 **저장된 진입선 근접/도달**(🎯 N차 도달,
1차 진입선까지 거리%) 표시. `GET /api/ai/watchlist/entry-points` 배치 조회 추가.

## 5. 화면 정합성 정리 + 심층 검증

- 대시보드 빠른시작 '관심종목추가'→'종목 발견', /watchlist/add AI발견 탭 제거(테마는
  /discover?q=로 런치), /discover ?q= 지원, 미사용 DiscoverTab 삭제·카피 갱신(164a110).
- **심층 검증**: 실제 Pro 계정 Firebase 세션을 브라우저(playwright)에 주입해 전 authed
  화면 순회 — 전 화면 렌더·콘솔에러 0 확인. 이전분석 카드·발견·watchlist 정리 모두 확인.
- **발견**: 분석 첫 진입 시 차트/시세 실패(Cloud Run 콜드스타트/버스트 5xx). CORS
  미들웨어는 이미 최외곽(정상)이라 플랫폼 레벨 5xx → **KIS REST 훅 백오프 재시도**로
  흡수(8047e80). 재배포 후 차트 정상 렌더 확인.

## 잔여/후속
- event 페르소나는 종목마다 여전히 flaky 가능(되면 풀분석, 안 되면 graceful). 근본
  안정화는 structured-output/tool-use 강제가 후보(별도).
- 분석 페이지 첫 로드 시 간헐 Cloud Run 503 콘솔 로그는 재시도가 흡수(사용자 무영향).
  더 줄이려면 min-instances 상향 등 인프라 튜닝(비용 대비 효익 낮아 보류).
